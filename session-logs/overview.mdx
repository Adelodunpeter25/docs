---
title: Session Logs
description: Capture terminal output for debugging, audits, and handoffs.
---

Session logging lets you keep a record of terminal activity when needed.

## Capabilities

- Start and stop logging per terminal session
- View saved logs inside FreeSH
- Open logs in dedicated tabs
- Delete logs you no longer need

## Suggested Workflow

1. Start logging before a critical operation
2. Run commands in terminal
3. Stop logging after completion
4. Review saved logs for verification or sharing

## Use Cases

- Incident response timelines
- Deployment change verification
- Team handoff notes

## Troubleshooting

### Log is empty or missing expected output

- Confirm logging was enabled before command execution
- Confirm session stayed active during capture window
- Confirm you are reviewing the correct session log
