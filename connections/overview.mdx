Use **Home > Connections** as your launchpad for day-to-day server work.

## What This Page Covers

- Creating a saved connection
- Editing host/auth details safely
- Organizing with groups
- Deleting stale entries

## Create a Connection

1. Open **Home > Connections**
2. Click **New Connection**
3. Enter required fields:
- Name
- Host
- Port (`22` default)
- Username
- Authentication method
4. Click **Save & Connect**

After first connect, the connection stays in your list for one-click reuse.

## Authentication Options

### Password

- Use when your host allows password login
- Password is stored in your system keychain

### Public Key

- Select a saved key, or paste/import a private key
- Add passphrase when required
- Passphrase is stored in your system keychain

## Edit Existing Connections

Use edit when host details change (hostname, user, auth method, key selection).

Good practice:
- Keep connection names stable (`prod-api-1`, `staging-db`)
- Encode environment in group, not only in name

## Grouping Strategy

Use groups when your list starts growing.

Example structure:
- `Production`
- `Staging`
- `Development`
- `Personal`

This keeps search and scanning fast when you have many hosts.

## Delete Behavior

Deleting a saved connection:
- Removes only local saved config
- Does **not** delete anything on remote servers
- Does not remove unrelated session logs/snippets automatically

## Troubleshooting

### Connection fails immediately

- Confirm host and port are reachable from your machine
- Confirm SSH daemon is running on target host
- Confirm firewall/security group rules allow inbound SSH

### Permission denied

- Re-check username
- Re-check selected auth method
- If key-based auth, verify key/passphrase pair is correct

### First-time host verification prompt appears

This is expected for new hosts.

- Compare fingerprint with a trusted source
- Accept only after verification
- The trusted key is then saved to Known Hosts
