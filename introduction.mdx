# FreeSH

FreeSH exists for one reason: remote work should stay fast even when you have many sessions open for hours.

I built it after hitting the same wall repeatedly with heavier tools: rising CPU, loud fan noise, and workflows that got locked behind cloud sync.

FreeSH is local-first, desktop-native, and focused on long-running terminal workflows.

## What FreeSH Is

FreeSH combines:

- SSH terminal sessions
- SFTP file operations
- Port forwarding
- Connection and key management

Everything runs locally. Your connections, history, and settings stay on your machine.

## Product Principles

- **Performance first**: startup and tab switching must stay responsive
- **Local ownership**: no mandatory cloud account for core usage
- **Secure defaults**: host-key verification and system keychain integration
- **Practical workflow**: built for people managing multiple hosts daily
