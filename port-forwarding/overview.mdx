Use port forwarding to route traffic securely through SSH.

## Supported Modes

- **Local forwarding**: local port -> remote host:port
- **Remote forwarding**: remote port -> local host:port
- **Dynamic forwarding**: SOCKS proxy via SSH

## Common Use Cases

- Access internal services from your machine
- Expose local development service to remote network
- Route browser/tool traffic through SOCKS

## Manage Tunnels

You can:
- create tunnel configurations
- start and stop active tunnels
- monitor status from the app
- auto-start selected tunnels after connection
